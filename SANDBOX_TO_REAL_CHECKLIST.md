# Чеклист: sandbox → real

Профиль бота: MOEX, dynamic portfolio (macro + MOEX/Finam), исполнение **adaptive**, ребаланс **10:00–18:10 МСК**, риск-лимиты как в `config.yaml`.

**Минимум в sandbox:** 4–6 недель непрерывной работы без ручных «костылей» в конфиге.  
**Уверенный вход в real:** 2–3 месяца sandbox + первый месяц real с **10–20%** планируемого капитала.

Отмечайте пункты: `[ ]` — не готово, `[x]` — ок.

---

## Еженедельная проверка (≈15 мин)

Проводить **раз в неделю** (удобно после воскресного отчёта в **18:00**).

### Telegram

| # | Проверка | Где / как | Ожидание |
|---|----------|-----------|----------|
| 1 | Робот жив | Сообщение «Робот работает» (раз в **24 ч**) | Приходит регулярно |
| 2 | Дневной дайджest | **18:00** | Equity, дневной PnL, просадка без сюрпризов |
| 3 | Авторебаланс | Сообщения ~**10:00** в окне | Без повторяющихся ошибок заявок |
| 4 | Алерты риска | Просадка **≥5%**, дневной убыток **≥2%** | Если были — робот **заблокировал** торговлю или вы понимаете причину |
| 5 | «Нет сделок» | Алерт после **72 ч** | Только если рынок/окно реально без сделок; иначе — разбор |
| 6 | Dynamic portfolio | Раз в **7 дн.** | Состав обновился или осознанно из кэша; не постоянный fallback на static |
| 7 | Выбор стратегий | Раз в **14 дн.** | Смена только при delta **≥0.08**; нет хаотичных `ai`/`deepseek` без OpenRouter |

Команды: `/status`, `/help` — equity, позиции, паузы, окно ребаланса.

### Дашборд и метрики (`:8000`)

| # | Проверка | Эндпоинт | Ожидание |
|---|----------|----------|----------|
| 8 | Health | `GET /health` | `broker_ok`, `config_ok`, `ready: true` |
| 9 | Статус | `GET /api/status` | `robot_running: true`, `trading_allowed: true` |
| 10 | Equity | `/api/status`, `/metrics` | `bot_equity` ≈ equity в status |
| 11 | Ошибки | `/metrics` | `bot_errors_total` не растёт неделями |
| 12 | График | `/api/equity` | Кривая без «обрывов» (VPS/брокер не падали надолго) |

### Логи (VPS или `data/logs/`)

| # | Файл | На что смотреть |
|---|------|-----------------|
| 13 | `bot.log` | `ERROR`, `config_error`, watchdog restart |

Проверяет: ERROR в логах, watchdog, «ребаланс без заявок при дрейфе», портфель в кеше, блокировки торговли, fallback dynamic portfolio.

## Накопительные критерии (к концу sandbox)

### Срок и объём данных

| # | Критерий | Порог |
|---|----------|-------|
| A | Непрерывная работа | **≥ 28 дней** (лучше **≥ 42**) |
| B | Исполненные сделки | **≥ 15–20** (не только расчёт ребаланса) |
| C | Полных циклов dynamic portfolio | **≥ 4** (7-дневный интервал) |
| D | Автовыборов стратегий | **≥ 2** (14-дневный интервал) |
| E | Автопереобучений (self-learn) | **≥ 3** (7-дневный интервал) |

### Поведение и риск (ваши лимиты из конфига)

| # | Критерий | Порог / действие |
|---|----------|------------------|
| F | Макс. просадка в sandbox | **≤ 12%** (блок торговли при **15%** — `risk.max_drawdown`) |
| G | Дневной убыток | Не было дней **>3%** без паузы (`risk.daily_loss_limit`) |
| G2 | Мягкий режим | При убытке **>2%** за день размер сделок уменьшался (`daily_loss_soft_limit`) |
| H | Kill-switch по индексу | При падении MOEX **≥5%** за день — пауза до конца дня (если такой день был — проверить) |
| I | Пауза по инструменту | После **3** убытков подряд — пауза **24 ч** сработала (если был кейс) |
| J | Размер позиции | Ни одна позиция не превышала **25%** equity (`max_position_pct`) |
| K | Разовая сделка | Не больше **10%** equity (`single_trade_max_pct`) |

### Качество стратегии

| # | Критерий | Как проверить |
|---|----------|---------------|
| L | vs buy&hold | Equity бота **не хуже** простого равновесного портфеля top-6 из `candidates` за тот же период |
| M | Комиссии | Оборот × **0.03%** учтён; частые мелкие ребалансы не «съедают» результат |
| D | Дрейф | Ребаланс по дрейфу **5%** не даёт лишних сделок каждый день (cooldown **60 мин**) |

### Инфраструктура

| # | Критерий | Порог |
|---|----------|-------|
| N | CI/CD | Deploy и Tests зелёные после последних изменений |
| O | Uptime VPS | Нет необъяснимых простоев **> 2 ч** в торговое окно |
| P | OpenRouter | Баланс **≥ $2** (`ops.openrouter_min_balance_usd`); macro не молчит неделями |
| Q | Finam (опционально) | Токен валиден; MOEX ISS работает и без него |
| R | Секреты | `.env` на VPS полный; `validate_config` без ошибок (`config_ok: true`) |
| S | Backup | `learned_params/params.json` бэкапится (`ops.learned_params_backup_interval_hours: 24`) |

### Стабильность конфига

| # | Критерий |
|---|----------|
| T | **2+ недели** без правок `config.yaml` «на каждый чих» |
| U | `dry_run: false` — заявки реально выставлялись |
| V | `ai_mode: false` — исполнение через adaptive (как задумано для боя) |
| W | Нет постоянного «Советники недоступны, static fallback» |

## Переключение на real (один раз)

### Конфиг

```yaml
mode: "real"
tinkoff:
  use_sandbox: false
  account_id: "<ID боевого счёта>"
```

В `.env` на VPS:

- `TINKOFF_TOKEN` — боевой токен (read + trade)
- `TINKOFF_ACCOUNT_ID` — боевой счёт
- Остальное без изменений (Telegram, OpenRouter, Finam)

### Рекомендуемые ужесточения на 1-й месяц real

Бот в `mode: real` **сам** ужесточает часть лимитов в коде; дополнительно можно вручную:

| Параметр | Sandbox | 1-й месяц real (опционально) |
|----------|---------|------------------------------|
| Капитал на счёте | 120k тест | **10–20%** от целевой суммы |
| `max_position_pct` | 0.25 | **0.15–0.20** |
| `single_trade_max_pct` | 0.10 | **0.05–0.07** |
| `rebalance_drift_pct` | 0.05 | оставить **0.05** |
| `ops.auto_start_sandbox` | true | **false** — старт только после осознанного `/start` в Telegram |
| `ops.exit_on_config_error_real` | true | оставить **true** |

### После деплоя real

- [ ] `GET /health` → `config_ok: true`
- [ ] В Telegram: **подтверждение real-торговли** (бот запросит)
- [ ] Первый ребаланс — вручную проверить заявки в приложении Тинькофф
- [ ] Первые **7 дней** — ежедневный `/status`, не еженедельный

## Первый месяц real (еженедельно + ежедневно)

### Ежедневно (2 мин)

- [ ] Дневной дайджest **18:00** — PnL и просадка
- [ ] Нет неожиданных алертов `config_error` / `no_trades`

### Еженедельно (как sandbox + дополнительно)

- [ ] Сравнить PnL sandbox vs real (поведение должно быть **похожим**, не идентичным)
- [ ] Проскальзывание: лимит **0.1%** от цены — заявки исполняются или висят?
- [ ] Комиссии в приложении брокера ≈ ожидание **0.03%** × оборот
- [ ] `bot_errors_total` стабилен

### Масштабирование капитала

Увеличивать долю на счёте только если **4 недели подряд**:

- [ ] Нет срабатывания `max_drawdown` / `daily_loss_limit`
- [ ] Нет ручных вмешательств в сделки
- [ ] Результат не хуже бенчмарка

Шаг масштабирования: **+25–50%** к текущему real-капиталу, не сразу весь объём.

## Красные флаги — в real **не** идти / остановить бота

- Просадка sandbox **> 12%** без понятной рыночной причины
- Постоянный fallback на static instruments
- `bot_errors_total` растёт каждый день
- Ребаланс каждый день с оборотом **> 30%** портфеля без необходимости
- OpenRouter / macro недоступны **> 1 недели** (состав портфеля «застыл»)
- Вы не можете объяснить последнюю сделку по `rebalance_decisions.log`
- Нет резерва на просадку: real-капитал = деньги, потеря **15%** которых критична
